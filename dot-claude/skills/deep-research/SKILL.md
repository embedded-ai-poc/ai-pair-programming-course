# DEEP-RESEARCH: Multi-Agent Research System

> **기반**: [Anthropic Multi-Agent Research](https://www.anthropic.com/engineering/multi-agent-research-system)
> **참고**: [199-biotechnologies/claude-deep-research-skill](https://github.com/199-biotechnologies/claude-deep-research-skill)

## Research Modes

| Mode | Phases | Duration | Use Case |
|------|--------|----------|----------|
| Quick | 3 | 2-5 min | 빠른 개념 파악, 초기 탐색 |
| Standard | 5 | 5-10 min | 일반적인 조사, 기술 비교 |
| Deep | 7 | 10-20 min | 복잡한 분석, 의사결정 지원 |

## 7-Phase Pipeline

```
[1.SCOPE] → [2.PLAN] → [3.SEARCH] → [4.VALIDATE] → [5.ANALYZE] → [6.SYNTHESIZE] → [7.REPORT]
                           ↓ 병렬
                    ┌──────┼──────┐
               [Agent-A] [Agent-B] [Agent-C]
               기술/공식   실무사례   비판/대안
```

---

## Phase 1: SCOPE (범위 정의)

**목표**: 조사 범위와 목적 명확화

**Actions**:
1. 사용자 질문/요청 분석
2. 핵심 키워드 추출
3. 조사 범위 설정 (기술적/비즈니스/사례 등)
4. 예상 결과물 형태 정의

**Output**:
```markdown
## Research Scope
- **Query**: [원본 질문]
- **Keywords**: [핵심 키워드 3-5개]
- **Scope**: [조사 범위]
- **Expected Output**: [결과물 형태]
```

---

## Phase 2: PLAN (조사 계획)

**목표**: 병렬 검색 전략 수립

**Actions**:
1. 3개 관점 정의 (기술/실무/비판)
2. 검색 쿼리 설계 (관점별 3개씩)
3. 신뢰할 소스 우선순위 설정

**Source Priority**:
```
1순위: 공식 문서, 학술 논문, 공식 블로그
2순위: 검증된 기술 블로그, GitHub repos, Stack Overflow
3순위: 커뮤니티 포럼, 개인 블로그, 뉴스 기사
```

---

## Phase 3: SEARCH (병렬 검색) ★ 핵심 ★

**Gate**: 3개 에이전트 병렬 실행 필수

**Parallel Agents**:

### Agent-A: Technical/Official
```
- 공식 문서, API 레퍼런스
- 학술 논문, 벤치마크
- 기술 사양, 아키텍처
```

### Agent-B: Practical/Case
```
- 실제 구현 사례
- 프로덕션 경험담
- Best Practices
```

### Agent-C: Critical/Alternative
```
- 한계점, 단점
- 대안 기술/접근법
- 비교 분석
```

**Each Agent Uses**:
- `WebSearch` - 웹 검색
- `WebFetch` - 상세 내용 추출
- `mcp__context7__query-docs` - 라이브러리 문서

---

## Phase 4: VALIDATE (소스 검증)

**목표**: 수집 정보의 신뢰성 검증

**Validation Checklist**:
- [ ] 소스 신뢰도 평가 (1-5)
- [ ] 정보 최신성 확인 (2년 이내 우선)
- [ ] 다중 소스 교차 검증
- [ ] 상충 정보 식별

**Output**:
```markdown
## Validated Sources
| Source | Type | Credibility | Recency | Key Finding |
|--------|------|-------------|---------|-------------|
| [URL]  | 공식 | 5/5 | 2025 | [발견 내용] |
```

---

## Phase 5: ANALYZE (분석)

**목표**: 수집 정보 종합 분석

**Actions**:
1. 패턴/트렌드 식별
2. 장단점 비교표 작성
3. 실행 가능한 인사이트 도출
4. 불확실한 영역 명시

---

## Phase 6: SYNTHESIZE (종합)

**목표**: 핵심 결론 도출

**Output Structure**:
```markdown
## Executive Summary
[핵심 결론 3-5줄]

## Key Findings
1. [주요 발견 1]
2. [주요 발견 2]
3. [주요 발견 3]

## Recommendations
- [권장 사항 1]
- [권장 사항 2]
```

---

## Phase 7: REPORT (보고서)

**목표**: 최종 결과물 생성

**Report Template**:
```markdown
# [조사 주제] Research Report

## TL;DR
[1-2문장 핵심 결론]

## Background
[배경 및 맥락]

## Methodology
[조사 방법 설명]

## Findings
[상세 발견 내용]

## Analysis
[분석 및 해석]

## Recommendations
[권장 사항]

## Sources
[참조 링크 목록]

---
*Generated by deep-research skill*
```

## Integration with Other Workflows

### Power-Workflow 연계
이 스킬은 power-workflow의 다음 Phase에서 AI 판단 하에 호출됨:

- **Phase 2 (ANALYZE)**: 요구사항 분석 시 관련 기술/사례 조사
- **Phase 4 (EXPLORE)**: 코드베이스 탐색 시 외부 패턴 조사
- **Phase 5 (DESIGN)**: 설계 시 아키텍처/라이브러리 비교

### 호출 방식
```
AI 판단 → Task tool로 deep-research 에이전트 호출
         → 결과를 상위 워크플로우에 반환
```
