# Verification Checklist

완전한 검증을 위한 체크리스트 레퍼런스

## 1. 요구사항 충족 (Requirement Compliance)

### 필수 확인
- [ ] 모든 명시적 요구사항이 구현됨
- [ ] 각 AC에 대응하는 코드가 존재
- [ ] 스코프 크립 없음 (요청하지 않은 기능 추가 없음)
- [ ] 기존 기능 손상 없음

### AC별 검증
```markdown
| AC | 구현 여부 | 위치 | 테스트 | 상태 |
|----|----------|------|--------|------|
| AC-XXX-1 | Y/N | file:line | Y/N | PASS/FAIL |
```

## 2. 테스트 품질 (Test Quality)

### 테스트 존재
- [ ] 각 AC에 해당하는 테스트 존재
- [ ] 테스트 파일이 올바른 위치
- [ ] 테스트가 실행 가능

### 테스트 충분성
- [ ] Happy path 테스트
- [ ] Error case 테스트
- [ ] Edge case 테스트
- [ ] Boundary condition 테스트

### 테스트 품질
- [ ] 의미 있는 Assertion
- [ ] 커버리지 게이밍 없음
- [ ] 독립적인 테스트 (상호 의존 없음)
- [ ] 명확한 테스트 이름

## 3. 코드 품질 (Code Quality)

### SOLID 원칙
- [ ] 단일 책임 원칙 (SRP)
- [ ] 개방-폐쇄 원칙 (OCP)
- [ ] 리스코프 치환 원칙 (LSP)
- [ ] 인터페이스 분리 원칙 (ISP)
- [ ] 의존성 역전 원칙 (DIP)

### 클린 코드
- [ ] 명확한 네이밍
- [ ] 적절한 함수 크기 (< 50줄 권장)
- [ ] 낮은 중첩 깊이 (< 4단계 권장)
- [ ] 코드 중복 없음 (DRY)
- [ ] 매직 넘버/스트링 없음

### 에러 핸들링
- [ ] 예외 처리 존재
- [ ] 적절한 에러 메시지
- [ ] 에러 로깅
- [ ] 복구 가능한 에러 처리

## 4. 보안 (Security)

### 인젝션
- [ ] SQL Injection 방어
- [ ] XSS 방어
- [ ] Command Injection 방어
- [ ] Path Traversal 방어

### 인증/인가
- [ ] 적절한 인증 체크
- [ ] 권한 검증
- [ ] 세션/토큰 관리

### 데이터 보호
- [ ] 하드코딩 시크릿 없음
- [ ] 민감 정보 로깅 없음
- [ ] 적절한 암호화

## 5. 성능 (Performance)

### 데이터베이스
- [ ] N+1 쿼리 없음
- [ ] 적절한 인덱스 사용
- [ ] 불필요한 쿼리 없음

### 메모리
- [ ] 메모리 누수 패턴 없음
- [ ] 큰 객체 적절히 해제
- [ ] 무한 루프 없음

### 프론트엔드 (해당시)
- [ ] 불필요한 리렌더링 없음
- [ ] 적절한 메모이제이션
- [ ] 번들 사이즈 적절

## 6. 문서화 (Documentation)

### 코드 문서화
- [ ] 복잡한 로직에 주석
- [ ] 공개 API 문서화
- [ ] 타입 정의 (해당시)

### 변경 기록
- [ ] 커밋 메시지 명확
- [ ] CHANGELOG 업데이트 (해당시)

## 판정 기준

### PASS 조건
- 모든 AC 구현 및 테스트됨
- Critical/High 보안 이슈 0개
- 커버리지 >= 80%

### PARTIAL 조건
- 대부분의 AC 충족
- Critical 0개, Warning 1-3개
- 커버리지 >= 70%

### FAIL 조건
- AC 누락
- Critical 또는 High 이슈 존재
- 커버리지 < 70%
